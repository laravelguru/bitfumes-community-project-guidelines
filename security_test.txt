o this end, you should protect the master branch to encourage code reviews before changes are merged and deployed. This will prevent work from being performed directly on master, and will foster collaboration among CoA developers.

In your repository navigation to Settings > Branches
Under “Protected Branches” select master
Enable the following settings:
Protect this branch
Require pull request reviews before merging
Dismiss stale pull request approvals when new commits are pushed
Include administrators
Some developers have (rightly) expressed concern about this review process potentially delaying critical fixes from going in in response to a fire drill. Fret not! Owners of the repository and the Organization it lives in can always force the merge even if the PR hasn’t been reviewed by a 3rd-party.
